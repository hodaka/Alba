# CLAUDE.md — このリポジトリでの運用ルール

Claude Code 向けの運用メモ。セッション開始時に自動で読み込まれる。
「レポジトリ参照して」と毎回言わなくても、ここを読めば前提を思い出せる。

## このリポジトリは何か

**Alba — 設定資料集**。TRPG／ゲーム制作のための世界観設定資料集。

## 応答の言語

- **日本語で応答する**（ユーザーは関西弁まじりのカジュアルな口調）。

## ディレクトリの役割

| ディレクトリ | 役割 |
|---|---|
| `stories/` | 一次ソース（本文）。**正史の唯一の正** |
| `characters/` | 二次資料（キャラ設定）。本文から導く |
| `world/` | 世界観・用語・命名規則・歴史 |
| `assets/` | 立ち絵・画像・プロンプト等 |
| `canon-notes.md` | 未確定・保留事項の置き場 |
| `楽屋/` | **非正史**。与太話・メタ・ボツ案・初期案（Gemini時代の設定など）。**インスピレーション用** |

> ## ⚠ **`楽屋/` は本編作業中は参照禁止**
>
> **`stories/` `world/` `characters/` `canon-notes.md` を書く・読む・考えるときに、
> `楽屋/` を開いてはならない。** 参考にするのも、根拠にするのも禁止。
> ＝**読まないことで、正史への混入を物理的に防いでいる。**
>
> **開いてよいのは、語り部が明示的に「楽屋を見よう」と言ったときだけ。**
> 楽屋の内容を正史へ引き上げられるのも、**語り部が指示したときだけ**（書記の判断では不可）。
> 詳細 → `楽屋/README.md`。

## 最重要原則

1. **本文（`stories/`）が正史の唯一の正。** 二次資料（characters / world）が本文と
   食い違ったら、常に本文を優先する。
2. **本文に無い設定を二次資料へ勝手に足さない。** 要約時の盛りすぎ・取り違えを防ぐため。
   キャラ設定や世界観は、必ず本文と突き合わせてから書く。
3. 確定していない裁定・保留メモは `canon-notes.md` へ。確定したら該当ファイルへ移す。

## 世界観の要点（迷ったら `world/` を参照）

- **民族名は短い形が正**：**フスク人（北）／ペルキ人（南）**。
- 古語「フスクス／ペルルキ」と現在語「フスク／ペルキ」は住人も無自覚に混在使用。
  セリフ・地の文でどちらが出ても誤りではない（現実の「摂津／大阪／梅田」の感覚）。
- **「アルバ」は意図的な多義語**。地名／緩衝地帯など複数の意味を持つ。出てきたら
  文脈でどの意味か見極める。追跡は `world/地名と民族.md` の該当節。
- **「アルバ」を冠する三代の傑物は本当に別々の三人**：**始祖アルバ（白狼）→ 百余年後に賢母アルバ
  （マッドアルバ）→ さらに十数年後に教祖アルバ**。**混同しない。** なお **「教祖アルバ」は他称で、
  本人の自称は「始祖アルバ」**（晩年に「三代はすべて自分」と唱えて正史を改竄した妄想ゆえ／全くの事実無根・
  一部の信者以外は取り合わない）。＝**本人の自称に引きずられて三者を同一視しないこと**
  （→ `world/教祖アルバ.md`「晩年」）。
- 詳細な歴史は `world/世界の成り立ち.md` `world/楽園の崩壊.md` を正とする。
- **【メタ・重要】魔法は “有る”＝これが正史**。本文『白い湯の秘密』で、ベルタが打ち上げられ **宇宙から大気圏
  再突入して生還する**＝魔法なしでは成立しない描写があり、**本文＝正史の唯一の正**（→ 最重要原則）ゆえ、
  **正史は「魔法有り世界」で確定**。＝**魔女は（少なくとも一部は）literal な魔法の使い手**。
  - **【定義・重要】「魔女」＝不死能力を持つ者＝ガイヤに罰せられた者。魔力・魔法の有無は条件ではない**
    （魔力ゼロの魔女が実在する＝クラウディ・クラウディア → `world/魔法.md`「魔女の定義」）。
    ＝**「魔法を使える女性＝魔女」と書かないこと。**
  - ただし設定は極力 **“魔法なしでも筋が通る”** よう組んである（ワクチン＝生物学、教祖アルバの神通力＝妄想 等）
    ＝**魔法を万能の逃げ道にしない**方針は維持。「魔法なし世界」は非正史の別分岐(AU)として温存してよいが、
    **正史ではない**（詳細 → `canon-notes.md`）。

## 作業のしかた

- 作業ブランチは指定に従う（**現状は `main` に直接コミットしている**）。
- 変更したら分かりやすいコミットメッセージで commit / push する。
- コミットメッセージの接頭辞は既存慣例に合わせる（例: `world:`）。

## 執筆セッションの進め方（基本モード）

**ユーザーは「語り部の長老」として世界を語る。Claude はそれを記録・整理する書記。**

1. **つけ足さない・改変しない。** ユーザーが語った内容を、勝手に盛らない・変えない・
   解釈で埋めない。あくまで語られたことだけを、整理しながら既存資料へ積み上げる。
2. 置き場所は既存のディレクトリ役割に従う（世界観→`world/`、本文→`stories/` など）。
   構成を整えるのは可。中身の創作はしない。
3. **語りの中で以下を見つけたら指摘する**（勝手に直さず、まず知らせる）：
   - 誤字脱字
   - 意味が通らない／不明瞭な箇所
   - 過去情報（既存資料）との矛盾
4. **一定量進んだら、キリのいいところで commit / push。** これを延々と繰り返す。
